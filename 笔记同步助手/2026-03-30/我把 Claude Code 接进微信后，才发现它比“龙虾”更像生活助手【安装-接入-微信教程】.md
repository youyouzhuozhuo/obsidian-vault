### 引言

我昨天收到一个读者提问，大意很简单：**Claude Code 到底怎么装？装完之后能不能像 OpenClaw 一样接入微信里用？**

我觉得这个问题可能是很多人的共识。其实它不只是一个简单的问题，而是一个更符合大家现在实际经验的选择。也就是说，如果我想把 AI 变成一个随手能用的工作助手，我应该选择的最稳妥工具或者路径是什么？

我的答案是：至少在我自己的使用体验中，Claude Code 依然是最稳妥、最靠谱的选择。这并不是因为它最通用，也不是因为它像 OpenClaw 一样更人性化，而是因为它更像一个真正能拿来推进工作的工具。

你给它一个目录，给它一个上下文，它更容易把你的事情往前做，而不只是简单的陪你聊几句天。**如果你把模型和聊天入口都配置好，它就可以变成从微信里随时能叫出来的本地助理。**

​

### 为什么我现在更愿意用 Claude Code

先说结论：

**我现在更愿意把 Claude Code 当主力，不是因为它最会表演，而是因为它更像一个能真正交付结果的工具。**

在我自己的体感里，它有三个优点。

-   第一，它的工作方式更像“进目录就开干”。
    

你不需要先理解整套系统架构，不需要先打许多系统命令和工具，也不需要解决各种安装配置问题。

只需要给它一个工作目录，它就可以读文件、理解上下文、修改内容并生成结果。

对于自媒体写文章、整理资料、搭建工作流这种真实任务来说，这种便捷性本身就是一种体验优势。

-   第二，它更适合重度使用。
    

很多 AI 工具第一次用都很惊艳，但你一旦开始天天用，真正决定体验的就不是“会不会”，而是“稳不稳”“顺不顺”“能不能连续做半小时以上的事”。Claude Code 在这件事上给我的感觉一直比较明确：它更像一个长期配合的工作伙伴，而不是一个新鲜的演示入口。

-   第三，它和本地文件夹的关系非常直接。
    

这一点非常重要。因为我不是只想让 AI 回答问题，我更想让它进入我的真实目录里做事。比如新建一个文件夹，帮我规划四月安排；

比如进入我的内容库，帮我起草文章；比如把现有材料整理成更结构化的版本。Claude Code 在这类任务上天然顺手。

​

### 为什么大家吐槽用不起来？

你直接走 Anthropic 官方模型链路，也会遇到两个非常现实的问题。

一是对国内用户来说，账号、网络、支付和稳定可用性经常会带来额外折腾。

二是如果你开始高频使用，高成本压力会很快从“可以接受”变成“每次调用都要想一下值不值”。而一旦你总想着省着用，Claude Code 这种本来应该拿来重度干活的工具，体验就会打折。

_今天我告诉你一个方法，就是从安装 Claude 入门，然后让它接入国内的大模型。通过使用低成本的方式，把 Claude Code 使使劲地用起来。_

​

### 第一步：先把 Claude Code 装起来

先说最重要的一句：

**Claude Code 的安装本身一点都不复杂，核心就是一条命令。**

请严格地把这条命令输入你的 Mac 电脑或者 Windows 电脑的命令行工具里面。

因为我使用的是 Mac 电脑，所以后续我这边的截屏和演示都以 Mac 电脑为准。

```
curl -fsSL https://claude.ai/install.sh | bash
curl -fsSL https://claude.ai/install.sh | bash
```

装完之后，用下面这条命令确认有没有成功：

```
claude --version
claude --version
```

如果能正常输出版本号，说明 Claude Code 已经进到你的系统里了。

**先安装 Claude Code，再把模型层配好，再正式开始用。**

​

### 第二步：不建议直接用 Claude 模型

Claude Code 当然可以直接走 Anthropic 官方模型，这也是它最原生的路径。但如果你的主要使用环境在国内，我并不建议把这条路当成默认首选。

原因主要有两个。

​

#### 第一，使用门槛并不只是“有没有账号”这么简单。

很多人以为装好了就结束了，实际上对国内用户来说，后面还会遇到网络、支付、账号风控、调用稳定性这些现实问题。你当然可以一个个去解决，但如果你的目标是“尽快把工具用起来”，这条路不一定是效率最高的。

​

#### 第二，价格会影响你真正的使用习惯。

Claude 的模型是出了名的贵。如果你把 Claude Code 作为日常工具，使用 Claude 的官方模型连续改稿写文章，或者持续写代码跑任务，这些模型的成本是非常昂贵的。

所以对我来说，更实用的选择不是“坚持原生”，而是“**保留 Claude Code 的交互方式和工作流，再把底层模型切到更适合自己的一套方案**”。

​

### 第三步：把国产大模型系列接进 Claude Code

\*\*很多人以为“Claude Code”就只能配“Claude 模型”\*\*，其实不是这样。其实国内的大模型基本上都支持 Claude Code，只需要查看并参考相应的官方文档进行接入即可。

我们使用的 Claude Code 只是一个工具的外壳，而真正作为“大脑”的模型本身，可以使用更便宜、实惠且实用的国产大模型。例如：

1.  DeepSeek
    
2.  MiniMax
    
3.  通义千问
    

这些模型都是可以接入的。因为我本人目前使用的是智谱 GLM-5 的 Coding Plan（代码套餐），每个月的价格相对来说比较实惠，所以在这里我将展示如何配置。

**你用的还是 Claude Code 这套终端交互和工作流，但底层真正跑的是国产大模型。**

![](https://pic.clipfx.app/b232765c2c8469e5115c175568a7190d.png)

![](https://pic.clipfx.app/295da85f2766174049991a4acd1a2f10.png)

到这里，其实 Claude Code 已经可以正常用了。

但如果你只停在这里，它依然只是一个“电脑上的命令行工具”。

我觉得真正让它变得更有意思的一步，是把它接到微信里。

​

### 第四步：微信不是官方功能，但社区已经把桥搭好了

**Claude Code 并没有官方直接提供“微信入口”这件事。**

今天能把它接到微信里，靠的不是 Anthropic 官方，而是社区的桥接方案。

我这次重点研究的项目是：cc-connect 把运行在你机器上的 AI Agent 桥接到你日常使用的即时通讯工具。代码审查、资料研究、自动化任务、数据分析，只要 AI Agent 能做的事，都能通过手机、平板或任何有聊天应用的设备来完成。

**把运行在你本机上的 AI Agent，桥接到你日常使用的聊天工具里。**

​

### 第五步：用 cc-connect 把 Claude Code 接到微信

`cc-connect` 的微信桥接，本质上不是把 Claude Code 搬到云端，而是把你本地这台机器上的 Claude Code 会话，接到一个微信入口里。

所以你要先理解它的工作方式：

**微信只是入口，真正干活的还是你本地目录里的那个 Claude Code。**

​

#### 1\. 先装对版本

这里有一个非常关键的坑，很多人第一次就会踩。

如果你想接的是**个人微信**，不能装稳定版的 `cc-connect`，因为官方安装说明和仓库 README 都明确写了：个人微信的 Weixin / ilink 支持，目前只在 `beta / pre-release` 版本里。

所以安装命令应该是：

```
npm install -g cc-connect@beta
npm install -g cc-connect@beta
```

装好后可以检查版本：

```
cc-connect --version
cc-connect --version
```

![](https://pic.clipfx.app/e859c3eef6760a47d23e9900eb475efa.png)

#### 2\. 或者直接让 Claude Code 帮你安装即可

**最简单的方式**，把这段话发给 Claude Code 或其他 AI 编码 Agent，它会帮你完成整个安装和配置过程：

```
请参考 https://raw.githubusercontent.com/chenhg5/cc-connect/refs/heads/main/INSTALL.md 帮我安装和配置 cc-connect
请参考 https://raw.githubusercontent.com/chenhg5/cc-connect/refs/heads/main/INSTALL.md 帮我安装和配置 cc-connect
```

![](https://pic.clipfx.app/5c9ea352473dbb055925ecc3a759d46e.png)

![](https://pic.clipfx.app/89f30eb46a72586a8ae41cd0fe82ecb9.png)

#### 3\. 用一条命令扫码绑定微信

![](https://pic.clipfx.app/863d2ce4bee2f7cd9010f309c8628c5b.png)

#### 4\. 首次接通后，先发一条消息

`cc-connect` 在微信桥接这条线上，会缓存一个 `context_token`。所以首次连接时，推荐顺序是：

1.  启动 cc-connect
    
2.  用已经允许的微信账号先发一条消息
    
3.  让系统完成第一次关联
    
4.  再继续使用 /new 或后续命令
    

如果你跳过这一步，很多人就会误以为“怎么已经配好了还是没反应”。

![](https://pic.clipfx.app/8e2320839d56e5287932c18d63a3bc73.png)

#### 5\. 记住这不是云机器人，而是远程操作本地 Claude Code

这一点我想单独强调一次。

很多人看到“微信连接 Claude Code”，脑子里会自动把它理解成“我在微信里有了一个无所不能的 AI 机器人”。这种理解不完全对。

更准确的说法应该是：

**你在微信里，多了一个远程入口，可以去调你本机上那个已经装好、已经配好模型、已经绑定好目录的 Claude Code。**

所以它真正的价值，不在于“能聊天”，而在于“能远程继续干活”。

​

### 连上微信之后，你可以拿它干什么

因为“能连微信”本身不是结果，**连上以后拿来干什么**才是结果。

在我看来，最值得写给普通读者看的，至少有两类。

​

#### 一类是编码和项目推进

如果你的 `work_dir` 指向某个代码项目，你可以在微信里直接说：

-   帮我看一下这个项目里登录模块的问题
    
-   到这个目录里新建一个页面草稿
    
-   检查一下 README 有没有过时
    
-   把今天改过的内容总结一下
    

它本质上是在远程调你本地那个 Claude Code 继续工作。

**有些本来要等你回到工位才能推进的事，现在可以先通过微信往前推一步。**

​

#### 另一类是进入指定文件夹做生活规划、写文章、整理资料

因为很多人一提到 Claude Code，先想到的是程序员。但如果你真的把它和目录结合起来看，它完全可以做很多“非传统编程”的任务。

比如你专门建一个目录，叫“生活规划”。

你可以在微信里直接说：

-   帮我新建一个四月计划文件夹
    
-   按工作、家庭、健康、财务四个维度，先起一个总纲
    
-   把下周待办整理成一份清单
    
-   基于我昨天写的记录，帮我生成一版复盘
    

再比如像我这种内容工作者，如果把它绑定到文章目录里，就可以直接说：

-   帮我新建一篇文章草稿
    
-   把这个选题扩成完整大纲
    
-   根据我现有语气，先写一个公众号版本
    
-   把文章里的步骤单独整理成截图清单
    

你会发现，**一旦微信只是入口，目录才是事实源，Claude Code 就不再只是“AI 聊天工具”，而更像一个能被你远程调度的本地助理。**

​

### 写在最后

**Claude Code 真正好用，不只是因为它安装简单，而是因为它能先成为你的主力工作工具，再进一步变成你随手能调出来的远程入口。**

安装它，本身不难。

**Claude Code + 国产模型 + cc-connect + 微信**

这样一来，我既保留了 Claude Code 这套很顺的本地工作流，又把成本、可用性和远程入口一起补齐了。
