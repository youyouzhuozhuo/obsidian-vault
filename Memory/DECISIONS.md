# 决策记录

## 公开仓库只放适合公开读取的内容

日期：2026-06-02

决策：GitHub 仓库使用公开模式，方便 GPT 读取和分析；敏感信息、日志、插件私密配置、附件和 token 不进入 Git。

原因：公开仓库能降低 AI 读取成本，但必须牺牲一部分隐私便利性。

## 第二大脑采用“原文 + 提炼层”

日期：2026-06-02

决策：保留原始笔记目录，同时新增 `MOC/` 和 `Memory/`，不要把全部内容混成一个大索引。

原因：原文适合查证，`Memory/` 适合 AI 快速读取，二者职责不同。

## 批量移动和删除需要确认

日期：2026-06-02

决策：AI 可以新增控制台、索引、摘要和脚本，但不要默认批量移动、删除、合并已有笔记。

原因：现有库由 Telegram、微信、坚果云和 GitHub 多端同步，批量操作风险较高。
